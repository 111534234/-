# 開發工具與技術選型說明

## 1. 總覽

本文件旨在列出「韓國首爾旅行網站」專案在開發、測試、部署各階段所需的軟體工具、函式庫與平台，並闡述選擇它們的理由，以確保團隊成員擁有一致的開發環境與技術共識。

---

## 2. 通用開發環境 (Development Environment)

| 工具 | 用途 | 選擇理由 |
| :--- | :--- | :--- |
| **IntelliJ IDEA** | 主要用於後端 Java/Spring Boot 開發 | 提供頂級的 Java 和 Spring 框架整合支援，內建強大的重構、調試和資料庫管理工具，能大幅提升後端開發效率。 |
| **Visual Studio Code** | 主要用於前端 React.js 開發 | 輕量級且擁有龐大的擴充套件生態系 (如 ESLint, Prettier)，對 JavaScript/TypeScript 支援度極高，是現代前端開發的主流選擇。 |
| **Git** | 版本控制 | 分散式版本控制系統的業界標準，能有效管理程式碼變更歷史，支援多人協作與分支管理。 |
| **GitHub / GitLab** | 程式碼託管與協作 | 提供遠端 Git 儲存庫、問題追蹤 (Issue Tracking)、程式碼審查 (Code Review) 和 CI/CD 整合功能，是團隊協作的核心平台。 |

---

## 3. 後端 (Backend)

| 工具/技術 | 用途 | 選擇理由 |
| :--- | :--- | :--- |
| **JDK 17 (LTS)** | Java 開發套件 | 執行和編譯 Java 程式碼的基礎。選擇 LTS (長期支援) 版本能確保穩定性與安全性。 |
| **Gradle** | 建置與依賴管理 | 專案已內建，它提供靈活的腳本來自動化編譯、測試、打包等建置流程，並能有效管理專案的外部函式庫依賴。 |
| **Spring Boot** | 核心應用程式框架 | 大幅簡化了 Spring 應用程式的初始設定與開發過程，內嵌 Web 伺服器，能快速建立可獨立執行的 RESTful API 服務。 |
| **PostgreSQL** | 關聯式資料庫 | 一款功能強大、開源且高度可靠的資料庫。支援複雜的查詢和 JSON 資料類型，非常適合需要確保資料完整性的應用。 |
| **Postman / Insomnia** | API 測試工具 | 開發後端 API 時，可在沒有前端介面的情況下，獨立測試各個端點的功能、效能與安全性，是 API 開發的必備工具。 |

---

## 4. 前端 (Frontend)

| 工具/技術 | 用途 | 選擇理由 |
| :--- | :--- | :--- |
| **Node.js & npm** | JavaScript 執行環境與套件管理器 | 執行前端開發伺服器、打包工具以及安裝各種前端函式庫的基礎。npm 是 Node.js 內建的套件管理器，生態系豐富。 |
| **React.js** | 使用者介面函式庫 | 由 Facebook 維護，是目前最流行的前端函式庫之一。其元件化的開發模式有助於建立可重複使用、易於維護的 UI。 |
| **React Router** | 前端路由管理 | 在單頁應用 (SPA) 中管理頁面之間的切換，讓使用者擁有像傳統多頁網站一樣的導覽體驗。 |
| **ESLint & Prettier** | 程式碼品質與風格統一 | ESLint 用於靜態分析程式碼以發現潛在錯誤，Prettier 則自動格式化程式碼。兩者結合能確保整個專案的程式碼風格一致，提升可讀性。 |

---

## 5. 部署與維運 (Deployment & DevOps)

| 工具/技術 | 用途 | 選擇理由 |
| :--- | :--- | :--- |
| **Docker** | 容器化 | 將應用程式及其所有依賴項打包成一個標準化的「容器」，確保在任何環境 (開發、測試、生產) 中都能一致地執行，大幅簡化部署流程。 |
| **雲端平台 (AWS, GCP, Azure)** | 應用程式託管 | 提供彈性的伺服器、資料庫、網路等基礎設施服務。可根據未來的使用者流量進行擴展，並提供監控、備份等維運功能。 |
