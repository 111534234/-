# 韓國首爾旅行網站：需求規格書 (REQ)

## 1. 專案概述

本文件旨在定義「韓國首爾旅行網站」的功能性與非功能性需求。此網站的主要目標是提供一個全面、易於使用的平台，讓計畫前往首爾的旅客能夠探索景點、規劃行程、預訂住宿與活動，並分享旅遊經驗。

## 2. 目標使用者

-   計畫前往首爾自由行的國際旅客。
-   尋找特定旅遊資訊（如美食、購物、文化體驗）的使用者。
-   希望規劃與分享個人化行程的旅客。

## 3. 功能性需求 (Functional Requirements)

### 3.1 使用者帳戶管理
-   **FR-001:** 使用者應能透過 Email 註冊新帳戶。
-   **FR-002:** 使用者應能使用帳號密碼登入系統。
-   **FR-003:** 使用者應能查看和編輯個人基本資料（如暱稱、頭像）。
-   **FR-004:** 系統應提供忘記密碼功能，允許使用者透過 Email 重設密碼。

### 3.2 旅遊資訊瀏覽
-   **FR-005:** 網站應以分類方式呈現首爾的旅遊資訊，至少包含：
    -   **景點介紹:** 如宮殿、博物館、公園等。
    -   **美食推薦:** 如餐廳、街頭小吃、咖啡廳等。
    -   **購物指南:** 如商圈、百貨公司、特色市集等。
    -   **住宿資訊:** 如飯店、民宿、青年旅館等。
-   **FR-006:** 每項資訊應包含詳細說明、地址、營業時間、交通方式、相關照片及使用者評價。

### 3.3 搜尋功能
-   **FR-007:** 使用者應能透過關鍵字搜尋網站內的旅遊資訊。
-   **FR-008:** 搜尋結果應能依據相關性、評價、距離等條件進行排序。
-   **FR-009:** 系統應提供進階篩選功能，例如依據區域、價格範圍、類別等篩選。

### 3.4 行程規劃
-   **FR-010:** 登入後的使用者應能建立、儲存及管理多個個人行程。
-   **FR-011:** 使用者能將網站上的景點、餐廳等資訊加入指定日期的行程中。
-   **FR-012:** 系統應能在地圖上視覺化呈現行程路線。
-   **FR-013:** 使用者應能將行程匯出或以連結方式分享給他人。

### 3.5 使用者評論與評分
-   **FR-014:** 登入後的使用者應能對景點、餐廳等項目發表評論及給予 1-5 星的評分。
-   **FR-015:** 使用者應能對他人的評論按讚或表示認同。
-   **FR-016:** 系統需顯示每個項目的平均評分。

### 3.6 預訂功能 (可為外部連結)
-   **FR-017:** 網站應提供住宿、門票或當地體驗活動的預訂選項。
-   **FR-018:** 預訂功能可導向至合作夥伴網站（如 Agoda, Klook）完成交易。

## 4. 非功能性需求 (Non-Functional Requirements)

### 4.1 效能 (Performance)
-   **NFR-001:** 網站主要頁面（首頁、分類頁）的載入時間在 3G 網路環境下應於 3 秒內完成。
-   **NFR-002:** 伺服器回應時間應在 500 毫秒以內。

### 4.2 使用性 (Usability)
-   **NFR-003:** 網站應採用響應式設計（RWD），在桌面、平板和手機上皆有良好的瀏覽體驗。
-   **NFR-004:** 網站介面應直觀、易於操作，符合現代網頁設計標準。
-   **NFR-005:** 網站應提供多語言支援（至少包含繁體中文、英文）。

### 4.3 安全性 (Security)
-   **NFR-006:** 使用者密碼需加密儲存。
-   **NFR-007:** 所有使用者資料傳輸過程應使用 HTTPS 加密。
-   **NFR-008:** 系統需具備防止常見網路攻擊（如 XSS、SQL Injection）的機制。

### 4.4 可靠性 (Reliability)
-   **NFR-009:** 網站應達到 99.5% 的上線時間。
